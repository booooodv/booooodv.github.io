GitHub上传文件

作者:booooodv    发布于: 2018-02-21

笔记说明

上传几次都需要查阅相关资料完成上传，所以还是决定写一个博客记录过程并方便以后查阅。主要记录命令行操作，所以OS X用户和windows用户是一样的。



导航：

  • 第一次操作GitHub仓库
  • 后续更新GitHub仓库

第一次操作GitHub

大致顺序是：
学习本地Git的基本使用—>注册GitHub账号—>创建GitHub工程—>本地git仓库相关操作—>上传本地代码

前期准备

Git基本使用：网上很多课程可以自行查找，我看的课程在我另外一篇博客创建博客里有提到可以跳转看一下。
注册GitHub账号：GitHub官网自行注册，如果看不到英文可以用google翻译翻译全页。

创建GitHub工程

本地Git

Git安装到官网上下载

  1. 命令行进入到相关目录生成本地Git仓库

    git init

  2. 然后把文件夹里所有文件添加到Git临时库中

    git add .

    . 表示所有文件，如果只要上传某个文件把.改为文件名

  3. 把临时库里所有文件放入Git仓库

    git commit -m "添加所有文件"

    “”写的是此次提交的备注

本地Git大概只有这几步，其他对Git仓库安装时一些指令应该在学习Git中进行操作的

上传到GitHub

与GitHub认证

  1. 在本地生成SSH KEY

    ssh-keygen -t rsa -C "youremail@example.com"

    引号写GitHub绑定邮箱，生成后会有地址提示，Windows在 C:\User\.ssh ，Mac就会在 User\.ssh ，找到 id_rsa.pub 文件打开复制里面内容

  2. 在GitHub认证
    进入GitHub主页点击右上角头像 —> Settings —> SSH and GPG keys —> New SSH key —> 在key中粘贴刚刚复制的内容

  3. 测试是否成功

    ssh -T git@github.com

    直接复制粘贴将会看到是否成功

绑定Git本地与工作仓库
git remote add origin https://github.com/name/test.git

网站其实就是你新建GitHub仓库网站，后面加上 .git

上传
git push origin master

如果是空文件请在 push 后面加上 -u

上传过程中可能会出现 failed to push some refs 错误，是因为本地不存在 README.md 文件，需要把两边仓库先合并使用代码

git pull --rebase origin master

在上传就不会有问题了

后续更新GitHub仓库（总结）

  1. 创建本地Git：

    git init

  2. 添加到临时库

    git add .

  3. 提交到Git库

    git commit -m "备注"

  4. 关联GitHub项目库

    git remote add origin 地址

  5. 上传到GitHub

    git push -u origin master
