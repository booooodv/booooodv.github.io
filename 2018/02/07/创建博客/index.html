创建博客

心路历程

在学习html、css过程中了解到全球最大同性交友网站GitHub 一直就觉得得记录些东西，但又不知道写在哪，所以对blog正好有需求，正好发现Github提供blog服务器，就顺势看网上教程一步步搭建起了自己的blog。

学习过程

用到技术：
  1. Git的安装和使用
  2. 域名管理
  3. Hexo的安装和使用
  4. MarkDown基本语法
  5. Html、Css（MD学习过程中）
学习途径：

博客搭建：GitHub+Hexo 搭建个人网站详细教程-知乎
Git学习：Git实用教程-网易云课堂
域名：域名注册-阿里云
MarkDown：Markdown 语法说明 (简体中文版)
Html、Css：HTML+CSS | WEB前端零基础实战课程-网易云课堂

遇到问题：
  1. 搭建时有些命令需要管理员权限，只要在前面加 sudo 输入密码就可以了
  2. 有些主题是真的用不了，如果只是局部乱码可以尝试 hexo clean 后再部署
  3. 主题很多功能都可以开关，甚至会Html、Css还可以改所有你想改的。
  4. 图床永远比本地存图更靠谱，更省内存。
  5. MarkDown # 后一定要跟空格，不知道为什么MarkDown编辑器不加空格显示变大，但是上传后无法识别。
  6. 域名租是年费而不是买断（天真认为买断差点买最贵的了）
  7. MarkDown我认为是每个人都应该学的，真的很好用。
  8. GitHub真的很多可以学习，也很有意思。
  9. hexo s 可以实时预览对网页的更改，不用总是部署然后看（我就是不断部署然后上网站上看）
总结经验：

Git大概花了一天时间，搭建Blog和MarkDown学习花了一整天时间，这样的连续学习让时间过得特别快，而且很愉快，感觉收获了很多，有目标的学习过程比较难得，而且整个过程并不是很难。搭建了Blog以后会让自己要更快的更新学习内容和总结，需要写更多GitHub和文章来更新Blog，虽然现在的文采不是很好，我相信通过Blog写下去，文采变好不是问题，我需要更多的阅读来充实大脑，更多的技术学习来更新GitHub，我相信Blog将见证我的成长，也不知道会不会有人看这些，就全当写给自己的话。我认为虽然微博、朋友圈早已取代博客，但是真的多久没有和自己对话了，所以博客变没有以前那样的利益性了，更多的是写给自己看，目标变得更加单纯。所以我认为博客还是有存在的意义，若你看到了这里，并且你也想搭建Blog，可以联系我，我将尽我所能帮助你。
