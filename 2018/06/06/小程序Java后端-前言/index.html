微信小程序Java后端-前言

作者:booooodv    发布于: 2018-06-06

前言

转眼间就有一个月没写博客了，最忙于开发微信小程序，参与前后端编写，但是主要负责用Java开发小程序后端。在网上找资源发现很少有用Java写的微信小程序后端，大多数是PHP写的，所以通过我不断的最近摸索和思考，基本完成后端的编写，所以准备写一个系列记录我用Java写小程序后端的一些难点和过程，部分代码会挂到GitHub上，希望能帮助一些只学了Java但是需要编写微信小程序后端的人一点参考。

技术

总结一下我用到的技术，也给想做的一个学习方向。

  1. JavaSE（用Java开发总得会基础知识）
  2. SSM框架（虽然最后用的是Spring—boot，但是其实大多数都是SSM的知识）
  3. SQL语句（只需要知道基础的一些语句）
  4. Linux基础（基础知识还是得知道的）
  5. 前端基础（总还是得知道一点前端的传输格式之类的）

框架

经过学习微信小程序的编写，发现小程序其实就是一个类似vue.js框架的前端页面，所以Java后端我主要学习了SSM（Spring+SpringMVC+Mybatis）框架来大家Java-web后端框架，但学完SSM并搭建以后发现并不是很适合小程序后端，因为小程序并不需要返回页面，只需要接收前端数据，然后加工处理并对数据库进行操作，再返回数据。

经过最后的查找资料，发现Spring-boot+Mybatis比较适合，所有最后选择这样的一个框架组合。

工作

写下在我认为我在后端所做的工作。

  1. 写接口负责接收前端信息（需要与前端合理沟通并统一格式）
  2. 设计数据库（与前端共同商量）
  3. 租服务器，买域名（当然是有报销的）
  4. Linux服务器环境搭建
